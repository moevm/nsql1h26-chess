# Circular Chess — запуск

Веб-приложение «круговые шахматы»: `backend` (Node.js + MongoDB) и `frontend` (статика + nginx).
Боты ходят только через публичный API по ключу.

## Запуск приложения

Из корня репозитория (нужен `.env` с `JWT_SECRET`, `MONGO_URI`, `PORT` — пример уже есть):

```bash
docker compose up -d --build
```

Поднимутся три сервиса:

| Сервис   | Адрес                         |
|----------|-------------------------------|
| frontend | http://localhost:8080         |
| backend  | http://localhost:3000/api     |
| db       | MongoDB (внутри сети compose) |

При первом старте БД заполнится тестовыми данными. 
- Вход админа: **`admin` / `admin123`**. 
- Вход для игроков **"имя_игрока" / `player123`**


## Запуск бота

**Боты загруженные через seed представлены тоько в качестве примера дынных, чтобы реально проверить работу бота, нужно следовать инструкции ниже**

Тестовый бот — это `bot.py` в корне репозитория. Он опрашивает API, находит свои активные партии и ходит случайным легальным ходом.

### 1. Получить API-ключ из приложения

Ключ выдаётся при создании бота:

- Зайдите в UI под админом → раздел **Боты** → создайте бота. После создания показывается `api_key` — **скопируйте сразу** (повторно он не отображается).
- Ключ можно перевыпустить кнопкой регенерации на странице бота.


### 2. Запустить бота в отдельном терминале

Через docker compose (профиль `bots`):

```bash
BOT_KEY=<API_KEY> docker compose --profile bots up --build bot
```

Контейнер бота ходит на backend по внутреннему адресу `http://backend:3000`, ключ берётся из `BOT_KEY`, период — из `BOT_INTERVAL` (по умолчанию 5 c).
